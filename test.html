Here are the steps you can use to assess and QA your Web documents. You will require Firefox and the following add-ons: Web Developper Toolbar, WAVE Toolbar. Other useful add-ons could include Firebug (Inspects CSS and code), ColorZilla / WCAG Contrast Checker, IE View Lite, HTML “Tidy” Validator, PDF Download and Link Evaluator (it checks for broken links but you must still test all your links).

1. Check for validation errors and validate to W3C standards

Instructions

  1. Tool Check page errors and accessibility


    Use the WAVE Toolbar

    • In the WAVE Toolbar, click on the red button Errors, Features and Alerts.
    • Correct any errors and restart this step until WAVE has detected no accessibility errors.
  2. Tool Use the W3C Validator
    • View the page’s source code, select ALL and COPY.
    • Go to http://validator.w3.org/check and select the Validate by Direct Input tab.
    • PASTE into the Direct input field and SUBMIT.
    • Correct any errors and restart validation. Repeat this step until you get this message:

3. Ensure there is no loss of content or functionality when the text is resized to 200%

Instructions

Manual check Pages need to be able to be resized text up to 200% without loss of content or functionality. In respective browsers, increase font size (not zooming of the page, just text). This also applies to text-based form controls (e.g. input boxes).

  • Firefox (2 steps):
    1. View > Zoom > Zoom Text Only
    2. View > Zoom > Zoom in (do it twice); Increase font size (200%)
  • IE: View > Text Size > Largest

Checking text-based form controls:

  1. Enter some text into text-based form controls that receive user entered text.
  2. Increase the text size of the content by 200%.
  3. Check that the text in text-based form controls has increased by 200%.

5. Ensure all content is keyboard accessible and there is no keyboard trap

Instructions

Manual check You need to ensure that you can navigate the page using only the keyboard:

  1. Identify all functionality on the content.
  2. Ensure focus is visible
  3. Check that all functionality can be accessed using only the keyboard.
  4. Ensure focus order preserves meaning (by using the TAB key)
  5. Check to see that keyboard focus is not trapped in any of the content.
    1. If keyboard focus appears to be trapped in any of the content, check that help information is available explaining how to exit the content and can be accessed via the keyboard.
  6. Check to see if there are any action taken on focus of an element. (e.g. Submiting the form on focus)
  7. For each instance of links and form elements in the source code, check that the name, value, and state are specified as indicated in the table of H91

6. Ensure all images has an alt, proper usage of images of text and contrast ratio

Instructions

  1. Manual check Manually determine the type of image, then inspect the Alt using the Web Dev Toolbar.

    Web Dev Toolbar: Images > Display Alt Attributes

    • Decorative
      • The Alt text is empty: alt="". An image that does not require a description. Visual enhancements, decorations or embellishments that provide no function or information beyond aesthetics to users who can view the images. They have no meaning in themselves and do not provide page content.
    • Simple
      • Standard image where the Alt is a short text description. They are images that are important to the content and convey information.
    • Image of text
      • Unless the image is a marketing item, can’t use images of text. Note: Criteria [1.4.5 Images of Text] does not apply to marketing images.
    • Complex
      • A form of long description must be provided, along with an Alt. If a (complete) description precedes or follows in the content, the Alt must specify accordingly. If no description exists, one must be added below the image, using the expand-collapse technique: descriptive text or data table. See CIC Web standards for full details on method.

        Expand/Hide:

  2. Tool Determine the contrast ratio between text and background

    Use the Color analyser tool and ensure that it passes level AA of contrast ratio.

    Note: Criteria “1.4.3 Contrast (Minimum)” does not apply to logos.
