Titre de la page

Est-ce que le titre décrit le contenu ou le but de la page web?

Details

Make sure the page has a clear Title. The page title is the information provided at the top of the browser.

The title needs to be descriptive and informative.

The page title is the first information assistive technology users receive when they load a web page. So it is important to have a title that describes the content of the page so users with assistive technology can confirm or not if they are looking at the right page.

It also helps users to navigate between the browser’s tabs if they have multiple tabs open.

The title of the page is the information that is usually provided in search results. Clear title helps users to find your page easier.

So by having a descriptive title, you help users with assistive technology, users with multiple tabs open and you improve the display of the results page.

Redimensionnement du texte

Est-ce que le contenu est encore lisible et fonctionnelle lorsqu’il est redimensionné à 200 %?

Details

  • Users with low vision might use the browser’s build-in feature to increase the size of the text.
  • We need to ensure that the page is still readable and functional when the text size is doubled.
  • Pages need to be able to be resized text up to 200% without loss of content or functionality.
  • When testing, make sure to increase the size of the text, not to zoom-in

Navigation avec le clavier

Est-ce que vous pouvez naviguer la page sans une souris?

Details

  • Many people do not use the mouse and rely on the keyboard only to interact with the web page
  • This requires keyboard access to all functionality of the page. This means that all functionality available via the mouse must be also available via the keyboard
  • While using the keyboard to navigate the content of page via tab key, arrows, enter key, space bar, etc., make sure the keyboard focus is visible and follow a meaningful order
  • When a page element receive focus or the user interact with an element or input information, it shouldn’t result in a change of the page such as activate a link or submit a form
  • While navigation through content of a page, make sure the keyboard focus doesn’t get trapped in any part of the document.

Images

Est-ce que toutes les images ont une version texte?

Details

  • Avoid images of text. If the same visual presentation can be made using text alone, don’t use an image. Unless the image is a marketing item and the layout is a part of the branding of a product.
  • There are 3 types of images: decorative, simple and complex images
    • Decorative image
    • Simple images
    • Complex images

Decorative images

  • are images that don't not require a description. It is a visual enhancements or decorations that provide no function or information beyond aesthetics to users who can view the images. They have no meaning in themselves and do not provide page content.
  • The Alt text for a decorative image must be empty: alt=""
  • an image with an Empty ALT will be ignored by screen readers
  • Decorative images can also be used via CSS. They will appear as background image and will not be captured by assistive technology

Simple image

  • are images that are important to the content and convey information.
  • They are standard images where the alternative version can be a short text description

Complex images

  • are images that contains lots of information. In this case, a long description and an alt must be provided.
  • Complex images are charts or diagrams for example
  • If the description precedes or follows the image inside the page, the Alt must specify this information.
  • The long description can also be available in another page.

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the background.

Custom Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


Reveal.addEventListener( 'customevent', function() {
	console.log( '"customevent" has fired' );
} );

Slide Backgrounds
